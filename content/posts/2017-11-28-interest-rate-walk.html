This Shiny app demonstrates the Cox-Ingersoll-Ross interet rate walk and an interest rate walk conducted using a bootstrap resampling technique. The code used to create this app is availale on GitHub. Assuming you have the necessary packages installed, you should be able to run this Shiny app on your local computer. A live version of the app is available here.

Bootstrap Resampling Method

This method takes a starting interest rate (selected by the user of the app) and applies randomly drawn historical treasury yield changes to that initial rate. The treasury yield changes can be drawn from either 1, 5, 10, 20, or 30 year treasuries (depending on the user’s selection). The user can also control the historical time period over which the yield changes are drawn from.

I am pretty happy with the output resulting from this method. Despite the method’s naivete, the plotted results look reasonable. One might have a hard time distinguishing the results of one of these walks from true interest rates over a given time period.

Cox-Ingersoll-Ross Walk

I used the sde::rsCIR() function to generate this walk. I chose to use the CIR parameters as they are described in the CIR Wikipedia Article which is slightly different from the sde::rsCIR(). I adjusted the parameters for use in sde::rsCIR() in the Shiny server.R file. This Shiny app’s inputs for this walk are:

  • a which controls the speed at which the interest rate reverts to the mean
  • b the mean interest rate
  • sigma which controls the volatility

No matter which paramters I select, the resulting walks seem far too concentrated around the mean and/or the yields bounce back and forth across the mean at an unconvincing fast frequency. It is very posisible that I am doing something wrong (This is the first time I have used the sde package), and I will investigate further when I have some time. The code is available on GitHub (link above); please reach out, if you see any errors.

Conclusion

Bootstrap resampling of historical treasury yield changes gives a far nicer quick eye test result than the CIR. Do you have a preferred method for conducting interest rate walks? That is probably not a question you hear every day :), but if you do, please let us know in the comments below.
